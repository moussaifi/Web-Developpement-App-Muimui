app.logger_setup¶

logger_setup.py customizes the app’s logging module. Each time an event is logged the logger checks the level of the event (eg. debug, warning, info…). If the event is above the approved threshold then it goes through. The handlers do the same thing; they output to a file/shell if the event level is above their threshold. :Example:

>>> from website import logger
>>> logger.info('event', foo='bar')
Levels:
  • logger.debug(‘For debugging purposes’)
  • logger.info(‘An event occured, for example a database update’)
  • logger.warning(‘Rare situation’)
  • logger.error(‘Something went wrong’)
  • logger.critical(‘Very very bad’)
You can build a log incrementally as so:
>>> log = logger.new(date='now')
>>> log = log.bind(weather='rainy')
>>> log.info('user logged in', user='John')
